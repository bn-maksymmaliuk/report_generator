## Генератор звітів

Сервер з REST API, котрий має ендпойнт(и) для керування процесом. Кількість та призначення ендпойнтів визначте самі.

Сервіс має мати змогу запустити генерацію кількох звітів одночасно (batch).

Для запуску звіту, треба задати джерело, тип звіту, спосіб його виводу.

Вхідні данні джерел для звітів мають бути провалідновані.

Його має бути легко дебажити (розуміти що відбувається і коли). 

У звітів можуть бути:

* різні джерела (база даних, відповідь від API, CSV, JSON)
* різні способи виводу (CSV, JSON, завантаження по API тощо)
* типів звітів (фільтрації) може бути багато

Виконайте наступне:

1. Спроєктуйте структуру коду (файлова структура та сігнатури), яка враховуватиме той факт, що таких генераторів звітів може бути запущено декілька паралельно, вони можуть мати відмінні одне від одного вхідні та вихідні джерела і вони мають виконуватись якомога швидше.
2. Достатньо описати по одній імплементації кожного елементу (візьмемо для прикладу вхід даних з файлу CSV та JSON як вивід, а звіт має бути по співробітниках).
3. Звіт має містити перелік співробітників, із заробітньою платнею більшою за 3500. У виводі мають бути присутні тільки ім'я та посада.

Приклад вхідних даних *CSV* та  формату виводу звіту в *JSON* дивіться у файлах `input_data.csv` та `output_example.json`, відповідно.
